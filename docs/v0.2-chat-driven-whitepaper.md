# v0.2 — 채팅 기반 백서 (Chat-Driven Whitepaper)

> 브랜치: `feat/chat-driven-whitepaper` · 베이스: `main`(v0.1.0 MVP)
> 한 줄 요약: **팀 채팅방을 그대로 쓰면, AI가 그 대화에서 백서를 길어 올린다. 떠드는 건 사람, 정리는 AI, 합의는 다 같이.**

이 문서는 v0.2 개발의 단일 기준(SSOT)이다. 진행하면서 결정·완료 항목을 여기 갱신한다.

---

## 1. 배경 / 동기

v0.1은 "절(section)을 골라서 글을 보내는" 다소 각진 입력 구조였다. 실제 팀은 슬랙/카톡처럼 **자유롭게 채팅하고 파일·링크를 공유**하며, 정작 아무도 쓰고 싶어하지 않는 결과물이 백서다. v0.2는 입력 표면을 **일반 채팅방**으로 바꾸고, AI가 그 채팅 내역에서 백서를 증류하는 구조로 전환한다.

핵심 명제는 그대로 유지된다: **대화 = 입력, 백서 = 출력.** 바뀌는 것은 입력의 형태(절 태깅 → 자유 채팅)와, 그 사이에 들어가는 **분류·관련도 계층**이다. 갈아엎기가 아니라 재구성이다.

## 2. 확정된 설계 결정 (2026-06-15)

| # | 결정 사항 | 선택 |
|---|---|---|
| 1 | **노이즈 ↔ 신호 분리** | **AI 자동** — AI가 관련도·절을 판정. (※ 2026-06-15 갱신: 채팅 내 핀/제외 교정 UI는 너무 번잡해 제거. "AI 비서" 모델 — 채팅은 단순, 분류·정리는 전부 뒤에서 자동. 교정은 향후 백서 화면에서) |
| 2 | **절 분류** | **AI 자동 분류** (교정은 향후 백서 화면 — 채팅 아님) |
| 3 | **증류 트리거** | **완전 자동** (※ 2026-06-15 갱신: 수동 버튼 폐기. 대화가 분류되면 AI가 알아서 해당 절을 재증류해 백서 갱신, 내용 시그니처 캐시로 중복 방지) |
| 4 | **파일·링크 범위** | 이번 버전: **업로드 저장 + 링크 보관 + 텍스트/링크는 AI가 읽어 근거로 사용.** 이미지·PDF 파싱은 다음 단계 |
| 5 | **입장 승인** | 채팅방(프로젝트)은 **프로젝트당 하나의 공용 방.** 입장 시 **승인 게이트** — 입장 요청 → 소유자(관리자) 승인 시 멤버로 합류. 기존 `invites`(소유자가 초대)에 더해 **입장 요청(join request) 방향**을 추가. 직군별 분리 채널/DM 없음 |

## 3. 기존 자산 재사용 매핑

| v0.1 자산 | v0.2에서의 역할 | 변경 |
|---|---|---|
| `blocks` (작성자·시각·내용·append-only) | **채팅 메시지** | "보낼 때 절 필수" 제약 제거 → 절은 선택/AI 부여 |
| `block_translations` (N직군 × N언어) | **메시지별 번역** (각자 직군·언어로 채팅 읽기) | 거의 그대로 |
| `section_content` (증류 산문) | **백서 출력** | 그대로 유지. 증류 *입력*만 변경 |
| `signatures` / 합의, `release_entries` | **합의·릴리스 스냅샷** | 그대로 |
| 4-렌즈 리더 | 대화=채팅방 / 백서=출력 / 나란히=채팅+문서 | 대화 렌즈를 채팅 UX로 확장 |

→ **불변식(잠금 블록 immutable, repo 단일 경로, 트랜잭션 보내기)은 유지.**

## 4. 데이터 모델 변경 (예정)

신규/변경 테이블 (멱등 마이그레이션으로 추가, 기존 제약·트리거 보존):

- `messages` — 채팅 메시지. (초기엔 `blocks`를 그대로 채팅 메시지로 사용하고, 절 필수만 완화하는 방향을 우선 검토. 별도 테이블 분리는 필요 시.)
- `message_relevance` — 메시지별 AI 관련도 판정 + 분류된 절(`section_key`) + 사람 교정값(pin/exclude). 컬럼: `message_id`, `ai_section_key`, `ai_relevance`(0~1 또는 enum), `pinned`(사람이 반영 강제), `excluded`(사람이 제외), `override_section_key`.
- `attachments` — 파일/링크. 컬럼: `id`, `doc_id`, `message_id`, `kind`(file|link), `url`/`path`, `mime`, `title`, `text_excerpt`(AI가 읽은 텍스트/링크 요약), `created_at`.
- `join_requests` — 채팅방 입장 요청. 컬럼: `id`, `project_id`, `user_id`, `requested_role`(직군), `status`(pending|approved|rejected), `created_at`, `decided_by`, `decided_at`. 승인 시 `project_members`로 합류. (기존 `invites`는 소유자→사용자 방향, 이쪽은 사용자→소유자 방향)
- 증류 입력 시그니처(`section_content.source_sig`)를 "해당 절로 분류·핀된 메시지 집합"으로 재정의.

> 정확한 컬럼/마이그레이션은 Phase 1에서 확정하고 이 표를 갱신한다.

## 5. UI 변경 (예정)

- **대화 렌즈 → 채팅방 UX**: 말풍선 타임라인, 입력창, 파일/링크 첨부, 메시지 리액션(👍/📌 "백서 반영"), 각자 직군·언어 번역 토글.
- **백서 뷰**: 절별로 "어떤 메시지에서 길어왔는지" 출처 표시 + 사람이 메시지를 절 사이로 드래그/재분류.
- **"지금 백서 갱신" 버튼**: 현재 채팅 기준으로 증류 1회 실행 → 백서 갱신 + 릴리스 스냅샷.
- 실시간 반영(폴링→라이브)은 별도 항목으로 검토(이 모델에선 중요도↑).
